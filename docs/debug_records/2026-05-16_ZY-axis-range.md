## 問題描述

- **現象**：無論直立還是平放，Z 軸（前後）與直立 Y 軸（上下）的可感知位移跨度都不太明顯，且 Z 軸在停手後會有輕微回彈。
- **預期行為**：Z/Y 軸的移動幅度應與 X 軸（左右傾斜）接近，讓使用者能直覺感受到三個方向的對稱控制範圍。
- **差異**：X 軸（重力傾斜，持續量）感覺明顯；Z 軸（線性加速度，瞬時量）感覺跨度小且不穩定。

---

## 根因分析

1. **Z 軸訊號的物理本質是瞬時加速度** → `worldAcc.y`（線性加速度）需要手部**持續加速**才有訊號，一旦停止加速（保持等速或靜止），訊號立即為 0，球歸回中心。這與 X 軸的重力傾斜（傾斜即有持續訊號）本質不同。**結果：確認（根本限制，非 bug）**

2. **平放 Z 軸死區過大（dead zone = 0.700）** → 嚮導測量靜止時的 Z 雜訊高達 0.700 m/s²，設置此死區後，輕微動作完全被過濾，只有明顯前後推動才有反應，導致「感覺跨度小」。**結果：確認**

   - Log 驗算：`[嚮導 Forward 平放] 最大幅=7.690  死區=0.700  可用範圍=6.990`
   - 死區佔最大幅 = 0.700 / 7.690 = **9.1%**，算合理但偏大

3. **直立 Y 軸 sensitivity 未被嚮導計算** → 保持預設值 0.300，未根據實際手勢幅度優化。**結果：確認（可調整）**

4. **Z 軸訊號幀間震盪劇烈** → 從 log 可見 0.5s 間隔內 Z 值從 +3.19 → -0.14 → 可能中間有更大的峰值，EMA 濾波後仍有大幅震盪，造成 scaled.z 的劇烈跳動（-5.76 → +1.88 → +5.99 in 1 秒）。**結果：確認（物理噪聲）**

**根本原因**
> Z 軸控制基於線性加速度（瞬時推力），而非重力傾斜（持續量），導致控制感與 X 軸截然不同；高雜訊加上大死區使可感知跨度進一步縮小。這是感測器物理特性的限制，非程式 bug。

## 修復方案

### 概念說明
此問題無法透過代碼根本解決，僅能透過**參數調整**改善體感。

### 建議參數調整（Inspector 手動調整，不修改代碼）

| 參數 | 目前值 | 建議值 | 說明 |
| ---- | ------ | ------ | ---- |
| 直立/平放 `Axis Scale Z` | 3.00 | 4.0 ~ 5.0 | 相同輸入獲得更大位移幅度 |
| 平放 `Input Filter Time` | 0.06s | 0.10s | 更強的 EMA 濾波，降低 Z 雜訊震盪 |
| 直立 `Sensitivity Y` | 0.300 | 嚮導重新測量 | Y 軸從未由嚮導優化 |

> ⚠️ 降低 `axisDeadzone.z`（死區）雖可讓 Z 更靈敏，但會引入更多靜止雜訊，不建議。

### 無程式碼變更

Z/Y 軸的「跨度不明顯」本質上是感測器信號類型決定的（瞬時 vs 持續），主要靠 `Axis Scale Z` 放大輸出補償。

## Lessons Learned

1. **成功做對**：從 log 的 `scaled_after.z` 逐幀追蹤，發現 Z 軸在 1 秒內可達 -5.76 到 +5.99 的劇烈震盪，確認為訊號噪聲問題而非程式邏輯錯誤。
2. **改進空間**：嚮導目前只優化 X 軸（MaxGesture）與 Z 軸（Forward），Y 軸 sensitivity 始終保持預設。應在嚮導中加入 Y 軸測量流程。
3. **預防措施**：設計 UI 時可明確標示「Z 軸為推力控制（需主動加速），X 軸為傾斜控制（傾斜即持續）」，讓使用者理解兩者的控制方式差異，避免誤以為是 bug。
