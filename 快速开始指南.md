# AR.js 位移追踪 - 快速开始指南

## 🚀 5 分钟快速上手

### 步骤 1：准备 Marker（1 分钟）

1. **下载 Hiro Marker 图案**
   - 访问：https://jeromeetienne.github.io/AR.js/data/images/HIRO.jpg
   - 或使用项目中的默认 Marker

2. **打印 Marker**
   - 使用 A4 纸打印
   - 尺寸建议：10cm × 10cm
   - 确保图案清晰

### 步骤 2：打开测试页面（30 秒）

1. 在手机浏览器打开 `index.html`
2. 点击顶部 **"📍 位移测试"** 按钮
3. 允许摄像头权限

### 步骤 3：开始追踪（30 秒）

1. 将 Marker 放在桌面上
2. 用手机摄像头对准 Marker
3. 等待检测成功（状态栏显示 "Marker: 已检测"）
4. 移动手机，观察数据变化

### 步骤 4：连接 Unity（2 分钟）

1. 启动 Unity 应用程序
2. 确保 WebSocket 连接成功
3. 将 `PositionController` 脚本附加到要控制的物体
4. 开始测试！

---

## 📱 操作说明

### 基本操作

| 操作 | 效果 |
|------|------|
| 手机向上移动 | Unity 物体向上移动 |
| 手机向右移动 | Unity 物体向右移动 |
| 手机向前移动 | Unity 物体向前移动 |
| 手机旋转 | Unity 物体旋转 |

### 控制按钮

- **重置初始位置**：将当前位置设为新的参考点
- **调整缩放**：放大/缩小移动幅度（默认 1.0）
- **设置偏移**：微调 Unity 中的物体位置

---

## ⚠️ 注意事项

1. **需要 HTTPS**：生产环境必须使用 HTTPS，开发环境可用 localhost
2. **光照要求**：确保 Marker 有充足且均匀的光照
3. **距离建议**：保持 20-50cm 的距离
4. **稳定性**：避免快速移动，保持手机稳定

---

## 🔧 常见问题快速解决

| 问题 | 解决方法 |
|------|----------|
| Marker 检测不到 | 改善光照、调整距离、确保 Marker 清晰 |
| WebSocket 未连接 | 检查网络、点击"重新连接"按钮 |
| Unity 物体不移动 | 检查 Unity 控制台、确认脚本已附加 |
| 位置方向相反 | 调整缩放为负值或修改 Unity 坐标映射 |
